# CAST Imaging ‚Üí AWS Transform Custom Power

## Overview

This power delivers end-to-end application modernisation in two distinct phases:

- **Phase 1 (Automated)**: Deep structural assessment using CAST Imaging MCP, data-driven probing to determine the optimal modernisation path, and generation of a fully-structured Phase 2 transformation definition. No code changes ‚Äî purely analytical.
- **Phase 2 (Execution)**: The generated TD is executed via AWS Transform custom with CAST Imaging MCP still available for verification, caller/callee analysis, and impact assessment during code transformation.

The separation ensures human oversight between analysis and execution ‚Äî the user reviews, edits, and approves the generated TD before any code is touched.

## Capabilities

### Phase 1: Assessment & TD Generation (CAST Imaging)

- **Application Discovery**: List all applications available in the CAST Imaging environment
- **Architecture Analysis**: Explore application architecture at multiple abstraction levels (layer, component, sub-component, technology-category, element-type)
- **Quality Assessment**: Review quality insights including CVEs, structural flaws, ISO-5055 violations, cloud readiness, and green/sustainability patterns
- **Transaction & Data Flow Analysis**: Understand application behaviour through transaction and data call graph exploration
- **Dependency Mapping**: Analyse inter-application and intra-application dependencies, packages with versions
- **Database Exploration**: Inspect database tables, columns, and data relationships
- **Code-Level Inspection**: Retrieve source code snippets, callers/callees, and violation locations with exact file paths and line numbers
- **Data-Driven Probing**: Ask targeted questions based on specific findings (not generic menus)
- **TD Generation**: Produce a fully-structured Phase 2 transformation definition with MCP verification guidance

### Modernisation Patterns Supported

There are 4 primary categories, each with distinct sub-patterns:

| Category | Sub-Patterns | Examples |
|----------|-------------|----------|
| **Technology Upgrade** | Language Version Upgrade | Java 8‚Üí17, Python 3.8‚Üí3.12, Node.js 16‚Üí22 |
| | Framework Version Upgrade | Spring Boot 2‚Üí3, Angular 14‚Üí18, Django 3‚Üí5 |
| | Framework Migration | Struts‚ÜíSpring Boot, jQuery‚ÜíReact, Express‚ÜíFastify |
| **Security Remediation** | CVE Remediation | Vulnerable dependency upgrades, code-level vulnerability fixes |
| | Structural Flaw Fixes | CWE violations ‚Äî SQL injection, XSS, resource leaks, error handling |
| **Code Refactoring** | Dead Code Removal | Unused methods, classes, stored procedures |
| | Complexity Reduction | Extract methods, decompose conditionals, simplify hotspots |
| | Coupling Fixes | Reduce inter-component dependencies, dependency inversion |
| | Idiom Modernisation | Legacy patterns ‚Üí modern language/framework idioms |
| **Cloud Readiness** | Cloud Blocker Resolution | Hardcoded configs, platform-specific APIs, stateful sessions |
| | State Externalization | Local files‚ÜíS3, sessions‚ÜíRedis/DynamoDB, cache‚ÜíElastiCache |
| | Containerization | Dockerfile, deployment manifests, health checks, env config |

An application may require a **combination** of sub-patterns across categories.

### Phase 2: Execution (AWS Transform Custom)

Phase 2 uses the TD generated by Phase 1. It:

- Takes the generated TD as its transformation definition
- Executes via AWS Transform custom (local or remote mode)
- **Requires CAST Imaging MCP access** during execution for verification, caller/callee analysis, and impact assessment
- Supports local execution (1-9 repos) or remote execution (10+ repos via AWS Batch/Fargate)

## Workflow

1. **Connect** ‚Äî Verify CAST Imaging MCP server is responding
2. **Select Application** ‚Äî List available applications, user picks one
3. **Deep Assessment** ‚Äî Gather comprehensive data: architecture, quality, dependencies, transactions, data flows, advisors (with tiered violation retrieval)
4. **Present Findings** ‚Äî Show quantified assessment summary with specific counts and locations
5. **Probe** ‚Äî Ask data-driven questions to determine modernisation path (never generic menus)
6. **Pattern-Specific Intelligence** ‚Äî Gather focused data for the chosen pattern (violations with locations, code snippets, file paths)
7. **Generate Phase 2 TD** ‚Äî Produce a fully-structured transformation definition with MCP verification guidance
8. **Handoff** ‚Äî Present the TD and instruct the user on reviewing and executing it

## MCP Server

This power uses the `presales-in` MCP server which connects to the CAST Imaging API. CAST Imaging MCP is required in **both phases** ‚Äî Phase 1 for assessment, Phase 2 for verification during execution.
